TypeScript

Cos'è:

TypeScript è un linguaggio di programmazione open-source sviluppato da Microsoft. È una versione tipizzata di
JavaScript, il che significa che aggiunge tipi statici al linguaggio, rendendo il codice più sicuro e
leggibile.

A cosa serve:

TypeScript serve a migliorare la manutenibilità e la robustezza del codice JavaScript, consentendo ai
programmatori di dichiarare tipi per le variabili, i parametri delle funzioni e le proprietà degli oggetti.

Che problema risolve:

Risolve problemi comuni in JavaScript come errori di tipo, comportamenti imprevisti e debugging complicato.
Aiuta anche a migliorare la collaborazione all'interno del team di sviluppo.

Il compilatore TypeScript

Perché è necessario:

TypeScript utilizza un compilatore per convertire il codice TypeScript in JavaScript, poiché i browser e
l'ambiente di esecuzione JavaScript nativo non comprendono direttamente il TypeScript.

Type Inference

Cos'è:

TypeScript è dotato di un sistema di inferenza dei tipi che può dedurre automaticamente il tipo di una
variabile in base al valore assegnato.

Utilizzi Pratici

Il tipo 'any':

Il tipo 'any' è utilizzato per rappresentare una variabile di cui non si conosce il tipo in fase di compilazione. È un tipo "escape" che consente di bypassare il sistema di tipizzazione di TypeScript. Tuttavia, il suo utilizzo dovrebbe essere evitato quando possibile, poiché può portare a errori di runtime.

Il tipo Union:

Il tipo Union consente di dichiarare una variabile con più tipi possibili. Ad esempio, `number | string` rappresenta una variabile che può essere di tipo numero o stringa.

Il tipo Tuple:

Una tupla è un tipo di dato in TypeScript che consente di definire un array di elementi con tipi specifici e un numero fisso di elementi. Ad esempio, [string, number] rappresenta una tupla con una stringa seguita da un numero.

Le Interfacce in TS:

Le interfacce sono utilizzate per dichiarare contratti o "forme" degli oggetti. Definiscono la struttura di un oggetto, inclusi i nomi e i tipi delle sue proprietà. Le classi possono implementare interfacce per garantire che soddisfino determinati requisiti.

Types vs Interfaces:

Entrambe le interfacce e i tipi (types) possono essere utilizzati per dichiarare forme di oggetti o tipi personalizzati. La principale differenza è che le interfacce sono estendibili (possono essere estese da altre interfacce) e possono essere implementate dalle classi, mentre i tipi sono più flessibili e possono rappresentare unioni, intersezioni e altri tipi più complessi.

I Generics:

I generics consentono di scrivere codice in TypeScript che è parametrico rispetto ai tipi. Questo permette di scrivere funzioni e classi che lavorano con vari tipi di dati in modo flessibile, senza dover specificare il tipo in anticipo. Sono indicati utilizzando il simbolo <T>, dove 'T' è una variabile di tipo generico. Ad esempio, Array<T> rappresenta un array generico in TypeScript.
